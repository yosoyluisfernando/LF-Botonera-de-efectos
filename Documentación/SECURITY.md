# Política de Seguridad / Security Policy

## Versiones Soportadas

Actualmente, solo la última versión estable (publicada en GitHub Releases) recibe actualizaciones de seguridad. Te recomendamos mantener tu aplicación siempre actualizada.

| Versión | Soportada |
| ------- | ------------------ |
| v1.1.x  | ✅ Sí              |
| < v1.1  | ❌ No              |

## Reportar una Vulnerabilidad

Por favor, **NO** abras un Issue público si descubres una vulnerabilidad de seguridad, ya que esto podría comprometer a todos los usuarios de la aplicación.

Canal recomendado:

1. Usa el reporte privado de vulnerabilidades de GitHub, si está disponible en este repositorio.
2. Si esa opción no aparece, contacta al mantenedor por un canal privado oficial indicado en el README o en el perfil de GitHub del proyecto.

Si crees haber encontrado un problema crítico de seguridad en **LF Botonera de Efectos** (por ejemplo, ejecución remota de código, escalada de privilegios a través de los atajos globales, exposición de archivos locales, o vulnerabilidades en el sistema de actualizaciones), repórtalo por uno de esos canales privados.

Incluye, si es posible:

- Versión afectada de la aplicación.
- Sistema operativo.
- Pasos para reproducir el problema.
- Impacto potencial.
- Cualquier prueba de concepto mínima que ayude a confirmar el problema sin poner en riesgo a otros usuarios.

Si el problema afecta a dependencias subyacentes como Tauri, Rodio, o Symphonia, evaluaremos la aplicación de un parche o actualización de la dependencia de inmediato.

Recibirás una respuesta en cuanto la vulnerabilidad haya sido analizada. Si el reporte se confirma, se preparará una corrección antes de publicar detalles técnicos de forma abierta.

---

*For English speakers: Please **do not** open a public issue for security vulnerabilities. Use GitHub private vulnerability reporting if available, or contact the maintainer privately through an official channel.*
